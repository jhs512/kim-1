# 05 — 폰 Gemini 지침 재작성 (제네릭, 처음부터)

Status: ready-for-agent

`docs/gemini-system-instructions.md`를 비서코드 하드코딩 없이 다시 쓴다. 이후 개인 인텔리전스 요청 사항에 반영(브라우저).

## 담을 내용
- **비서 선택**: "비서 {코드}"로 활성화, 세션 유지, 안 대면 되묻기, "비서코드 뭐야?"→활성 코드. 코드 하드코딩 금지.
- **파일명 구조**: `{비서코드}_{no}_{namespace}_{doctype}_{visibility}_{title}` — 슬롯별 의미, 폴더 무시·이름 타깃팅, `{비서코드}_` 아닌 문서 무시.
- **조회·순회 최대화**: 진입 노드에서 끝내지 말고 엣지(target_doc_name)를 적극 순회해 서브그래프로 답. weight 높은 것 우선.
- **Drive 거부 방지**: "너는 Drive를 조회/생성할 수 있다. '못 한다'고 말하기 전에 반드시 먼저 시도하라."
- **never-refuse → 큐**: 직접 능력(지메일 읽기/캘린더 CRUD/Drive 읽기·생성/recall) 밖 요청은 거부 대신 `{비서코드}-calendar`에 `⏳[대기]` 작업 등록(요청 명시)하고 "큐에 넣었다"만 알림.
- **작업 상태 조회/정보보충**: "작업중 뭐야/정보필요 뭐 있어" → 캘린더 작업 상태 요약. 사용자가 정보 주면 해당 작업 `필요정보` 채우고 `대기`로.

## 수용
- 지침 전문에 특정 비서코드(kim-1 등) 하드코딩 0.
- 6개 항목(비서선택/파일명/순회/Drive거부방지/never-refuse/상태조회)이 명확한 지시문으로 존재.
- 개인 인텔리전스 "요청 사항"에 섹션별로 반영(기존 kim-1 항목 대체).
