# 04 — 워커 루프 + 작업종류 디스패치 (kim-work 스킬)

Status: ready-for-agent

kim-1 클로드 코드가 큐를 드레인하며 상태머신을 돌린다. 스킬 `kim-work`(한글).

## 동작
1. `bash scripts/kim-cal.sh due`로 만기 `대기` 작업 목록.
2. 각 작업: `set 작업중` → 작업종류 판별 → 수행 → `성공`/`실패`/`정보필요`(+이력) → 반복이면 `reenqueue`.
3. 작업종류 → 동작:
   - 메일 발송/답장 → gws gmail (발송은 **사용자 승인** 후; 승인 필요하면 `정보필요`).
   - 문서 수정 → gws drive/docs.
   - 지식 저장 → `kim-learn` 증류 → `kim-sync`.
   - 길게 조사 → 리서치 후 결과를 작업 `이력:`/설명에 보고.
   - 판별 불가/정보 부족 → `정보필요`(무슨 정보가 왜 필요한지 명시).
4. 자동화: `/loop` 또는 `schedule`(cron)로 주기 실행. 각 작업은 독립 — 하나 실패해도 다음 진행.

## 규칙
- `실패` 자동 재시도 없음. `성공`/`실패` 캘린더 유지.
- 외부 부수효과(메일 발송 등)는 **사용자 승인 게이트** 존중(안전 규칙).
- 모든 전이 이력 기록.

## 실증
큐에 "OO에게 메일 보내줘" 등록 → 워커가 작업중→(승인)→성공, 이력 확인. `정보필요`건에 정보 붙여 `대기`로 → 재개.
