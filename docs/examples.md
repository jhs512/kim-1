# 김비서 사용 예시 (~100)

발화 → 김비서 동작/답. `[직접]`=폰이 바로, `[큐]`=할 일로 적어두고 컴퓨터 일꾼이 처리,
`[상태]`=큐 상태 조회, `[정보]`=정보 보충, `[반복]`=반복 예약, `[선택]`=비서 선택.
비서코드는 예시로 `kim1`을 쓴다("비서 my55"면 `my55_` 시트·`my55-calendar` 사용).

## 비서 선택 · 기본 [선택]
1. "비서 kim1" → (활성 비서코드를 kim1로 설정)
2. "비서코드 뭐야?" → "kim1 입니다."
3. (아무 것도 안 정하고) "그거 찾아줘" → "어느 비서요?" (되물음)
4. "비서 my55로 바꿔" → 이후 my55_ 시트·my55-calendar 사용
5. "너 뭐 할 수 있어?" → 직접 가능(메일 읽기·일정·드라이브·조회)과 큐로 처리하는 일 안내
6. "지금 무슨 비서 쓰고 있어?" → "kim1"
7. "비서 kim1, 오늘 뭐부터 할까?" → 오늘 일정 + 밀린 할 일 요약
8. "안녕 김비서" → 인사 + 무엇을 도울지

## 지식 조회 · 순회 [직접]
9. "복리가 뭐였지?" → kim1_ 시트 검색해 '복리' 노드로 답
10. "복리 공식 알려줘" → 복리 노드에서 defined 엣지 따라 '복리 공식' 노드까지 순회해 답
11. "단리랑 복리 차이?" → 두 노드 + 연결(대비) 관계로 설명
12. "그때 우리가 정한 거 뭐였지?" → decisions 노드 검색
13. "이 주제로 내가 뭐 아는 거 있어?" → 관련 서브그래프(노드+이웃) 요약
14. "72법칙 노드 근거 어디서 왔어?" → 출처(derived_from) 엣지로 추적
15. "복리랑 연결된 거 다 보여줘" → 이웃 노드들(엣지 타입까지)
16. "이거 관련해서 더 깊이 아는 거?" → weight 높은 엣지 우선 순회
17. "내 지식 중에 금융 관련 뭐 있어?" → tags에 finance 있는 노드
18. "그 개념 요약만" → 노드 summary
19. "그거 신뢰도 얼마짜리야?" → 노드 confidence
20. "관련 노드 없으면 없다고 해" → 없으면 "kim1에 없다" + (일반 지식)으로 보충 표시
21. "이거 저장돼 있어?" → 검색해서 있으면 노드, 없으면 없다고
22. "이 노드에서 두 다리 건너 뭐 있어?" → 2홉 이웃 순회

## 지식 저장 [큐]
23. "이거 기억해둬: 복리는 장기일수록 유리" → 큐에 '지식 저장' 작업 등록, "큐에 넣었어요"
24. "이 내용 저장소에 넣어줘" → 큐 등록(정리는 일꾼이 노드로 증류)
25. "방금 대화 요점 저장해" → 큐 등록
26. "이 링크 정리해서 지식으로" → 큐 등록(요청에 링크)
27. "이거 결정으로 기록해" → 큐 등록(decision 노드 예정)
28. "이 사실 추가해줘" → 큐 등록(fact 노드 예정)
29. "메모 하나 남겨: 다음 분기 목표 재검토" → 큐 등록
30. "이 사람 연락처 지식에 넣어" → 큐 등록(contact 노드 예정)
31. "오늘 배운 거 저장" → 큐 등록
32. "이 아이디어 나중에 발전시키게 저장" → 큐 등록

## 메일 [읽기=직접, 발송·답장=큐]

34. "김대리한테 온 메일 찾아줘" → [직접] 검색해 보여줌
35. "오늘 중요한 메일만 추려줘" → [직접] 요약
36. "이 메일 뭐라고 왔어?" → [직접] 내용 읽어줌
37. "김대리한테 답장해줘: 내일 3시 좋아요" → [큐] '메일 답장' 작업 등록
38. "거래처에 견적 요청 메일 보내줘" → [큐] 등록(발송은 승인 후)
39. "이 메일 전달해줘" → [큐] 등록
40. "안 읽은 뉴스레터 정리해서 나한테 메일로" → [큐] 등록
41. "답장 초안 써서 큐에 넣어" → [큐] 등록
42. "스팸 같은 거 있나 봐줘" → [직접] 읽어 판단
43. "이 스레드 요약해줘" → [직접] 읽어 요약

## 드라이브 [읽기·생성=직접, 수정=큐]
55. "그 문서 찾아줘" → [직접] 검색
56. "이 폴더에 뭐 있어?" → [직접] 조회
57. "회의록 새로 하나 만들어줘" → [직접] 새 문서 생성
58. "이 내용으로 메모 파일 만들어" → [직접] 생성
59. "그 문서 내용 읽어줘" → [직접] 읽기
60. "기존 기획서 이 부분 고쳐줘" → [큐] '문서 수정' 작업 등록
61. "이 문서에 문단 추가해줘" → [큐] 등록

66. "이 주제 깊게 리서치해서 정리해줘" → [큐]

## 작업 상태 조회 [상태]
73. "지금 진행중인 일 뭐야?" → [작업중] 작업들 요약
74. "정보필요로 멈춘 일 뭐 있어?" → [정보필요] 작업 + 뭐가 필요한지
75. "밀린 할 일 몇 개야?" → [대기] 개수
76. "오늘 끝난 일 뭐야?" → [성공] 작업들
77. "실패한 일 있어?" → [실패] 작업 + 이유
78. "그 조사 어떻게 됐어?" → 해당 작업 상태·결과
79. "큐에 뭐 쌓여 있어?" → 상태별 요약
80. "메일 보내는 거 처리됐어?" → 해당 작업 상태
81. "지금 뭐 하고 있어?" → [작업중] 요약
82. "다 끝났어?" → 남은 [대기]/[작업중] 유무

## 정보필요 보충 [정보]

84. "아까 그거 답장 내용은 '수요일 좋아요'로" → 해당 작업 정보 보충 후 [대기]

## 능력 경계 · "안돼 금지" [큐]
101. "이거 (폰이 직접 못 하는 일) 해줘" → "안 된다" 대신 큐에 넣고 "큐에 넣었어요"
